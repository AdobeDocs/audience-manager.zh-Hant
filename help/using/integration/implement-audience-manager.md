# 實作 Audience Manager {#implementing-audience-manager}

本節概述並說明與Audience Manager資料管理平台入門相關的流程([!DNL DMP])。 本部分旨在幫助業務團隊、項目經理和技術經理瞭解Audience Manager實施過程。 開始Audience Manager可能需要大約六週到三個月的時間，具體取決於您的資料收集需要。

我們的實施技術有助於與新客戶建立咨詢性夥伴關係。 此流程旨在：

* 瞭解和瞭解您的業務需求；
* 制定一個可操作的計畫，以滿足這些要求；
* 開發定制解決方案，幫助滿足獨特的需求或使用案例；
* 確保您的專有資料已導入並在Audience Manager中可用。

我們的合作夥伴解決方案和客戶管理團隊將在實施過程之前、期間和之後與您密切合作。

Audience Manager採用分階段的方法來設定和實施。

## 構建、Test和培訓階段 {#build-test-train-phase}

在構建、test和培訓階段，您將與指定的合作夥伴解決方案負責人一起查看資料收集策略和原型。

<!-- c_build_test_train.xml -->

您的資料收集策略將進行端到端 [!DNL QA] 測試。 合作夥伴解決方案將跟蹤發現的錯誤並與我們的系統工程師協調問題解決。 客戶培訓可以與這些其他工作同時開始。

下表介紹了在此階段進行的關鍵活動：

| 活動 | 用途/說明 | 參與者 |
|---|---|---|
| 準備資料收集策略 | 與Adobe技術團隊合作，構建滿足您業務要求的資料收集計畫 | 業務和技術團隊 |
| 部署和test代碼 | 在試運行環境中test建議的解決方案並執行跨瀏覽器測試 | 技術團隊 |
| 驗證功能並解決錯誤 | 檢查並傳遞結果、解決錯誤並重新test | 技術團隊 |
| 用戶培訓 | 提供有關Audience Manager功能、工具和報告的教育和瞭解 | 業務團隊 |

此階段的交付項可包括：

* 已完成和接受的資料收集計畫；
* 端到端 [!DNL QA] 測試；
* Audience Manager用戶介面基本功能說明；
* 接受和簽名。

## 代碼實施 {#code-implementation}

Audience Manager使用Data Integration Library([!DNL DIL])，接收觀眾資料。

有關如何使用的詳細說明 [!DNL DIL]，請參閱 [獲取和實施DIL代碼](../dil/dil-overview.md)。

### 參與者

合作夥伴解決方案可以直接與您的技術團隊協作，幫助部署代碼、解決最終問題並滿足其他要求。

## 實施後支援 {#post-implementation-support}

我們的協作工作不會止於最終部署。 實施完成後，我們的客戶管理團隊將接手。

客戶經理在產品實施過程完成後提供持續的支援和咨詢服務。 您可以期望與客戶經理定期會面。 這些會議確保您從Audience Manager中獲得最大的使用量和價值。

聯繫我們 [這裡](https://www.adobe.com/products/audiencemanager.html) 的子菜單。
