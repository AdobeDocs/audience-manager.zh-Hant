# Implementing Audience Manager {#implementing-audience-manager}

This section outlines and explains the processes related to getting started with the Audience Manager data management platform ([!DNL DMP]). 本節旨在協助商業團隊、專案經理和技術經理瞭解Audience Manager實作程序。根據您的資料收集需求，Audience Manager快速入門大約需要周到個月。

我們的實施技術有助於與新客戶建立諮詢合作關係。此程序旨在：

* 探索並瞭解您的業務需求；
* 製作可採取行動的計劃，以滿足這些需求；
* 開發自訂解決方案以協助符合獨特的需求或使用案例；
* 確保您的專屬資料已匯入並提供至Audience Manager。

我們的合作夥伴解決方案與帳戶管理團隊將會在實施程序之前、期間和之後與您密切合作。

Audience Manager採用分階段的設定與實施方式。

## Build, Test, and Train Phase {#build-test-train-phase}

在建置、測試和培訓階段中，您將會使用指定的合作夥伴解決方案來檢閱資料收集策略和原型。

<!-- c_build_test_train.xml -->

Your data collection strategy will undergo end-to-end [!DNL QA] testing. 合作夥伴解決方案將追蹤發現的錯誤，並與我們的系統工程師協調問題解決方案。客戶培訓可與這些其他努力同時開始。

下表說明在此階段中發生的重要活動：

| 活動 | 目的/說明 | 參與者 |
|---|---|---|
| 準備資料收集策略 | 與Adobe技術團隊合作建立符合您業務需求的資料收集計劃 | 商業與技術團隊 |
| 部署和測試程式碼 | 在測試環境中測試提議的解決方案，並執行跨瀏覽器測試 | 技術團隊 |
| 驗證功能並解決錯誤 | 檢查並傳達結果、解決錯誤並重新測試 | 技術團隊 |
| 使用者訓練 | 提供Audience Manager功能、工具和報告的教育與認識 | 商業團隊 |

此階段的交付項目可包括：

* 完成且接受的資料收集計劃；
* End-to-end [!DNL QA] testing;
* Audience Manager使用者介面功能的基本指示；
* 接受及簽署。

## Code Implementation {#code-implementation}

Though the deployment process may seem complex, the code implementation is as simple as adding a few lines of [!DNL JavaScript] adjacent to the closing `</body>` tag of your website.

<!-- c_code.xml -->

### 部署工作

The Audience Manager code snippet calls [!DNL Akamai] to download the business rules set up previously in the user interface. 此外，用戶端瀏覽器會快取此項資訊，以協助減少頁面和伺服器的載入時間。我們的程式碼和資料收集方法旨在維護庫存的使用者體驗。

### 參與者

合作夥伴解決方案可直接與您的技術團隊合作，協助部署程式碼、解決最終疑慮並滿足其他需求。
