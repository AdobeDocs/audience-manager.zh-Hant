# 字彙表{#glossary}

進一步閱讀的定義和連結。

## A-B {#a-b}

**演算法模型**

Use [!UICONTROL Algorithmic Modeling] as a means of extending reach beyond the core of users you&#39;ve identified. 此功能可協助您透過自動化資料分析發掘新的獨特受眾。Manage your [!UICONTROL Algorithmic Models] in **[!UICONTROL Audience Data > Models]**.

See [Understanding Algorithmic Models](../features/algorithmic-models/understanding-models.md#understanding-models).

<br> 

**BaAAM**

[!UICONTROL Bulk Management Tools]。[!UICONTROL Bulk Management Tools] in in [!DNL Audience Manager] are a Microsoft Excel-based of tools that allows you calling、modify or delete multiple objects at一次with a single operation.您可以使用資料來源、衍生的訊號、目的地、資料夾、區段和特徵。The feature uses a Microsoft Excel spreadsheet with macros that make secure, authenticated calls to the [!DNL Audience Manager] APIs.

See [Bulk Management Tools](../reference/bulk-management-tools/bulk-management-intro.md).

## C-D {#c-d}

**CDF**

[!UICONTROL Customer Data Feed]。[!UICONTROL CDF] 檔案代表大量下載的資料，可讓 [!DNL Audience Manager] 您在使用者介面所施加的限制之外處理 [!DNL Audience Manager] 資料。[!UICONTROL CDF] 檔案包含 [!DNL Audience Manager] 與事件呼叫( `/event`)傳送至我們伺服器的相同資料。這包括使用者ID、特徵ID、區段ID，以及事件呼叫所擷取的所有其他參數等資料。

See [Customer Data Feeds](../features/cdf-files.md).

<br> 

**CRM ID**

CRM ID是客戶在自己的CRM系統中識別使用者的ID。我們使用Audience Manager中的詞語DPUUID，而不是CRM ID。

See DPUUID in the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**客戶可定址對象**

In [Addressable Audience](/help/using/features/addressable-audiences.md), this metric represents devices that:
* 在回顧視窗中已實現規則型或已登錄特徵
   **AND**
* 不論同步時間為何，都能與所選目的地同步。

<br> 

**客戶屬性**

See [Customer Attributes](https://marketing.adobe.com/resources/help/en_US/mcloud/attributes.html) in the [!DNL Experience Cloud Core Services] product documentation.

<br> 

**客戶匹配率**

客戶可定址對象*「客戶總觀眾」以百分比表示。See [Addressable Audience](/help/using/features/addressable-audiences.md).

<br> 

**客戶總觀眾人數**

In [Addressable Audience](/help/using/features/addressable-audiences.md), this metric represents a count of devices that have realized either a rule-based trait on your properties or an onboarded trait from your offline files during the look-back window.

<br> 

**demdex.net**

Demdex.net is a legacy domain controlled by [!DNL Adobe]. It reflects [!DNL Audience Manager]&#39;s original, pre-acquisition name ( [!DNL Demdex]). [!DNL Adobe] 於2011 [!DNL Demdex] 年收購，並重新品牌公司 [!DNL Audience Manager]。`demdex.net` 對網域的所有HTTP呼叫都是傳送到的呼叫 [!DNL Adobe]。

請參閱[瞭解向 Demdex 網域進行的呼叫](../reference/demdex-calls.md)。

<br> 

**DAID**

[!UICONTROL Device Advertising IDs] 是唯一的裝置識別碼，用來識別行動裝置。這些ID由裝置製造商指派，而非由Adobe指派。We support iOS and Android device IDs in [!DNL Audience Manager].

See the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**目標**

In [!DNL Audience Manager], a destination is any other system (ad server, DSP, ad network, etc.) 您要與其共用資料。The [!UICONTROL Destination Builder] in our UI provides the tools that let you create and manage these data delivery processes. [!DNL Audience Manager] 目的地功能 **[!UICONTROL Audience Data > Destinations]** 位於其中。

<br> 

**DIL**

The [!UICONTROL Data Integration Library] is an API library used by [!DNL Audience Manager] to collect user interaction data. See [Data Integration Library (DIL) API](../dil/dil-overview.md).

<br> 

**dpm**

[!UICONTROL Data Provider Match]。It tells internal [!DNL Adobe] systems that a call from [!DNL Audience Manager] or the ID service is passing in customer data for synchronization or requesting an ID. 請參閱[瞭解向 Demdex 網域進行的呼叫](../reference/demdex-calls.md)。

## E-F {#e-f}

**Experience Cloud ID (ECID)**

Previously named the [!DNL Marketing Cloud] ID (MID or MCID). [!DNL Experience Cloud] ID是ID服務的核心。這是您網站訪客的唯一和永久性識別碼。See Cookies and the [Experience Cloud ID Service](https://marketing.adobe.com/resources/help/en_US/mcvid/mcvid_cookies.html).

<br> 

**資料夾特徵**

自動分組檔案夾分類法內的特性。階層中的每個資料夾都會自動建立特徵，用來定義區段。

See [Folder Traits: About](../features/traits/about-folder-traits.md).

<br> 

**頻率Capping**

廣告商想要將特定創意顯示給一般使用者的次數上限。You can configure various frequency capping expressions in [!UICONTROL Segment Builder].

See [Recency and Frequency](../features/segments/recency-and-frequency.md).

## G-H {#g-h}

**GAID**

Google Advertising ID是Google指派給執行Android作業系統之硬體裝置的唯一裝置ID。See the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**GUID**

全域唯一識別碼的縮寫字元。We don&#39;t use the term GUID in [!DNL Audience Manager]. In our case, the GUID is the [!DNL Audience Manager] UUID.
See [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

## I-J {#i-j}

**IDFA**

廣告商識別碼，Apple指派給其產品的唯一裝置ID。See the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**傳入**

The process by which you can send audience data from other sources to [!DNL Audience Manager]. See [Sending Audience Data](/help/using/integration/sending-audience-data/send-audience-data.md).

<br> 

**整合代碼**

When working with the [!DNL Audience Manager] UI or API, you have the option of adding an integration code when creating traits, segments, or data sources. 在這些情況下，整合代碼有不同用途：

* [!UICONTROL Traits]：整合代碼是ID、SKU或內部業務程序使用之其他值的欄位。選填。
* [!UICONTROL Segments]：整合代碼是使用者定義ID或其他公司特定資訊的欄位。選填。
* [!UICONTROL Data Sources]：當您想要建立跨裝置資料來源、使用Experience Cloud ID服務或搭配使用時，需要整合代碼 [!UICONTROL Profile Merge Rules]。See [Create a Data Source](../features/manage-datasources.md#create-data-source) for more information.

See [Algorithmic Modeling](../reference/aam-glossary.md#a-b).

## M-N {#m-n}

**MCID**、 **MID**

See the [Experience Cloud ID](../reference/aam-glossary.md#e-f).

## O-P {#o-p}

**PCS**

[!UICONTROL Profile Cache Server]。The [!UICONTROL PCS] is a large database, running on Apache Cassandra. It stores data received for active users from server-to-server transfers and the [!UICONTROL DCS]. [!UICONTROL PCS] 資料包含裝置ID、已驗證的描述檔ID及其相關特徵。

See [Data Collection Components](../reference/system-components/components-data-collection.md).

<br> 

**個人資料連結**

See [Profile Merge Rule Options Defined](../features/profile-merge-rules/merge-rule-definitions.md).

<br> 

**描述檔合併規則**

[!UICONTROL Profile Merge Rules] 可讓您控制資料 [!DNL Audience Manager] 用於劃分的類型。

See [Profile Merge Rule Options Defined](../features/profile-merge-rules/merge-rule-definitions.md).

## Q-R {#q-r}

**實現**

網站上訪客符合特徵的動作。You can use the [Visitor Profile Viewer](../features/visitor-profile-viewer.md) tool to obtain information on trait realization by a specific user.

## S-T {#s-t}

**區段**

區段(或對象)是一組共用一般屬性的使用者。

See [Segments: Purpose, Composition, and Rules](../features/segments/segments-purpose.md).

<br> 

**區段可定址對象**

In [Addressable Audience](/help/using/features/addressable-audiences.md), this metric represents the number of users who have belonged to the segment during the report look-back period and have an active ID sync on your site. Segments can include your own first-party data and second party and third party data, via traits acquired in the [Audience Marketplace](/help/using/features/audience-marketplace/audience-marketplace.md).

<br> 

**區段總人口數**

In [Addressable Audience](/help/using/features/addressable-audiences.md), this metric represents a count of all the devices that were a member of your segment during the report look-back period.

<br> 

**區段匹配率**

群體可定址讀者訓練圖總計人口族群總計為%。See [Addressable Audience](/help/using/features/addressable-audiences.md).

<br> 

**訊號**

Signals are the smallest data units in [!DNL Audience Manager] and are expressed as key-value pairs.

See [Signals, Traits, and Segments](../reference/signal-trait-segment.md).

<br> 

**特性**

特徵是一或多個訊號的組合。See [Signals, Traits, and Segments](../reference/signal-trait-segment.md).

<br> 

**特徵人口族群**

See [Trait and Segment Population Data in Segment Builder](../features/segments/segment-builder-data.md).

**TTL(Time-to-Live)**

TTL定義了合格訪客留在特徵中的天數。TTL設定於特徵而非區段上。如果訪客在TTL間隔之前未看見符合資格的特徵，則會離開區段。Read more in [Segment and Trait Time-to-Live Explained](/help/using/features/traits/segment-ttl-explained.md).

<br> 

## U-V {#u-v}

**UUID**

[!DNL Audience Manager] 唯一使用者ID。See the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**Visitor ID**

[!DNL Experience Cloud] ID服務(先前的訪客ID)提供通用的永久性ID，可識別所有解決方案中的訪客 [!DNL Experience Cloud]。

See the [Experience Cloud ID Service](https://marketing.adobe.com/resources/help/en_US/reference/marketing-cloud-id-service.html) documentation.
