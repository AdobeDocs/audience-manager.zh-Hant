# 2019 年文件更新 {#documentation-updates}

2019 年發佈的新文件或修訂文件清單。包含 Experience Cloud 發行說明中可能未涵蓋的 Audience Manager 文件小幅更新。

如需瞭解 [!DNL Audience Manager] 和其他 [!DNL Experience Cloud] 解決方案的變更，請參閱 [Experience Cloud 發行說明](https://experienceleague.adobe.com/docs/release-notes/experience-cloud/current.html)。如需舊版發行說明，請參閱[先前年份](../docs-updates/docs-2018.md)區段。

## 2019年11月 {#aam-november-updates}

| 主題 | 說明 |
|--- |----|
| [加州消費者隱私權法案 (CCPA) 支援與隱私權文件編修](../overview/data-security-and-privacy/data-privacy.md) | 2020年1月1日生效的[加州消費者隱私法(CCPA)](https://www.caprivacy.org/about)將為加州居民提供有關其個人資訊的新權利，並對在加州經營業務的特定實體施加資料保護責任。<br> <br> Audience Manager可協助您透過Adobe Experience Platform隱私權服務等隱私工具，處理資料存取和刪 [除請](https://docs.adobe.com/content/help/zh-Hant/experience-platform/privacy/home.translate.html) 求，以遵守隱私權法規所規定的義務。<br><br> 我們已更新目 [前的退](../overview/data-security-and-privacy/data-privacy-requests.md#opt-out-requests) 出管理程式，加入選擇退出任何宣告ID（例如CRM ID）。若已宣告ID選擇退出，則已宣告ID和最後連結的裝置將會選擇退出Audience Manager資料收集。 退出請求功能也會以批次和即時傳送等方式，將取消細分請求傳送給支援此功能的[目標合作夥伴](../overview/data-security-and-privacy/assets/AAM-Partners-October2019.xlsx)。 <br><br> 此外，我們已重新設計 [資料安全性](../overview/data-security-and-privacy/data-security.md)、 [資料隱私權](../overview/data-security-and-privacy/data-privacy.md)和 [資料](../overview/data-security-and-privacy/data-governance.md) 控管檔案，讓您更輕鬆找到遵循上述法規所需的資訊。 |

## 2019年3月 {#aam-march-updates}

| 主題 | 說明 |
|--- |----|
| [全域資料來源](../features/global-data-sources.md) | 所有 Audience Manager 客戶均可使用全球資料來源，內含 Apple、Samsung、Microsoft、Roku 和 Android 裝置製造商所產生的裝置廣告 ID。這些 ID 由製造商提供，用於廣告目的。您可以使用全球資料來源同步裝置 ID，並匯入或匯出這些對應所中斷的資料。Audience Manager 會根據格式，驗證客戶匯入的裝置廣告 ID (DAID)，確保符合裝置製造商列出的標準格式。 |
| [ID 對應限制](../features/administration/usage-limits.md#id-mapping-limits) | 我們已限制與裝置 ID 相關的跨裝置 ID 對應的數量，藉此改善區段劃分、報告和匯出。新增的限制有助於緩解不當整合及不精確的資料集對擷取和下游處理所產生的影響。<br/> 新限制為: <ul><li>每個跨裝置 ID 100 個裝置 ID</li><li>每個裝置 ID 10 個跨裝置 ID</li><li>1000 個裝置 ID 對 1 個裝置 ID</li></ul> |
